树的直径

树的直径 是 树中长度最长的路径，可以使用树型 \(DP\) 求解

​ 任选一个点作为无向树的根节点，将这个根节点拎起来（可以想象为节点是有质量，边是没有质量的），考虑从集合的角度划分问题：

  • 任何一条路径 都可以划分到其 路径最高节点 对应的集合

  • \(dp[u]\) 表示以 \(u\)（\(u = 1,2,...,n\)）为 路径最高点 的所有路径下的最长路径

如下图，红色线便是 \(u\) 作为制高点的一条路径（同时也正好是 \(u\) 集合下的一条最长路径）

​ 考虑上图，我们已知了 \(dp[2]\) 和 \(dp[3]\)，如何使用树形 \(dp\) 的思想求解 \(dp[1]\)？我们可以发现：

  • \(dp[1]\) 和 \(dp[2]\)、\(dp[3]\) 之间并没有直接关系
  • 但 以 \(1\)​ 为制高点的最长路径：
    • 要么选择其下方 最长的一条路径 （此时次长路径的长度 < 0）
    • 要么选择其下方 每一个儿子方向上的路径最长的两条 （此时次长路径的长度 > 0）

所以我们可以存储 每个点往下能走的最长路径（记作 maxlen），那么我们可以得到转移方程：

\[ dp [u] = max _1\left\{ \max len\left( son_i \right) +w\left [ u \right] \left [ son_i \right] \right\} +max _2\left\{ \max len\left( son_i \right) +w\left [ u \right] \left [ son_i \right] \right\} \] 其中，\(max_1\)​ 表示最大值，\(max_2\)​ 表示次大值，\(maxlen\) 显然是可以递归的，可以使用 \(dfs\) 求解，即 \[ maxlen(u)= max(maxlen [son_i]+w [u][son_i]) \] 最终答案即为 \(dp\)​ 数组中的最大值

注意：

  • 假设顶点 \(u\) 有 \(t\) 个儿子节点 \(son_1\),..., \(son_t\)，则 \(max_1\)、\(max_2\) 是从含有 \(t+1\) 条路径的集合 { \(u\)，\(u\)-> \(son_1\)->...，\(u\)-> \(son_2\)->...，······，\(u\)-> \(son_t\)->...} 中选出最长的 \(d_1\) 和次长的 \(d_2\)
    • 路径 \(\{u\}\) 的长度为 \(0\)，因此我们将 \(max_1\)、\(max_2\) 的初值赋为 \(0\)

    表示如果从当前节点 \(u\) 往每一个儿子方向上的 \(t\) 条路径长度最大值/次大值小于 \(0\)，我们就 不 从当前这个点往下走，这样路径即为 \(\{u\}\)，距离为 \(0\)

    • 剩下的 \(t\) 条路径是当前节点 \(u\) 往下走，每一个儿子方向上的路径，从第 \(2\) 个点开始就各不相同，所以这 \(t\) 条路径也一定是完全不同的路径
    • \(max_1\) 一定是从当前顶点 \(u\) 向下走的最大距离，但是 \(max_2\) 不一定是从当前顶点 \(u\) 向下走的次大距离，因为最大距离和次大距离对应的路径有可能经过 \(u\) 的同一个儿子节点，但是我们的集合不包含这种情况

最近公共祖先 LCA

​ 给定一棵有根树，若节点 \(z\) 既是节点 \(x\) 的祖先，也是节点 \(y\) 的祖先，则称 \(z\) 是 \(x\)、\(y\) 的公共祖先。在 \(x\)、\(y\) 的所有公共祖先中，深度最大的一个称为 \(x\)、\(y\) 的最近公共祖先，记为 \(LCA(x,y)\)​

\(LCA(x,y)\) 是 \(x\) 到根的路径与 \(y\) 到根的路径的交会点。它也是 \(x\) 与 \(y\)​ 之间的路径上深度最小的节点。求最近公共祖先的方法通常有三种：

预处理

\(BFS\) 预处理得到 \(depth[\ ]\)、\(fa[\ ][\ ]\)

  • \(depth[i]\) 表示 \(i\) 的深度，\(1\le i \le n\)​
  • \(fa\left[ i,j \right]\) 表示从 \(i\) 开始向上走 \(2^j\) 步到达的节点，\(0\le j \le logn\)

\[ \begin{cases} fa\left [ i,0 \right] = i\ \text{的父节点}\\ \\ fa\left [ i, j \right] = fa\left [ fa\left[ i, j-1 \right] , j-1 \right] , 1\le j\le logn\\ \end{cases} \]

  1. 由于 \(2^j=2^{j-1}+2^{j-1}\) ，因此从 \(i\) 向上跳 \(2^{j}\) 步到达 \(fa[i,j]\)，等价于先从 \(i\) 向上跳 \(2^{j-1}\) 步到达 \(fa[i,j-1]\)，再从 \(fa[i,j-1]\) 向上跳 \(2^{j-1}\) 步最终到达 \(fa[i,j]\)
  2. 这类似于一个树型 \(DP\) 过程，“阶段”就是节点的深度。因此，我们可以对树进行广度优先遍历，按照层次顺序，在节点入队之前，计算它在 \(fa[]\) 中相应的值
  • 哨兵 \(0\) 号节点：\(depth[0]=0\)，如果从 \(i\) 开始跳 \(2^j\) 步会跳过根节点，那么 \(fa[i,j]=0\)
  • 时间复杂度：\(O(nlogn)\)

查询

基于 \(depth[\ ]\)、\(fa[\ ][\ ]\) 计算 \(LCA(a,b)\)​

  • 算法思想：
  1. 先将两个点跳到同一层
  2. 让两个点同时往上跳，一直跳到它们的 最近公共祖先的下一层

为什么要跳到最近公共祖先的下一层，而不是直接跳到最近公共祖先那一层呢？

  • 实现步骤：
  1. 不妨设 \(depth[a] \ge depth[b]\)（否则可交换 \(a\)，\(b\)）
  2. 用二进制拆分思想，把 \(a\) 向上调整到与 \(b\)​ 同一深度

具体来说，就是依次尝试从 \(a\) 向上走 \(k=2^{logn},…,2^1,2^0\) 步，检查到达的节点是否比 \(b\) 深。在每次检查中，若是，则令 \(a=fa[a,k]\)

  1. 若此时 \(a=b\)，说明已经找到了 \(LCA\)，\(LCA\) 就等于 \(b\)（这就是上面的图中的第三种情况）

  2. 用 二进制拼凑 的思想，把 \(a\)，\(b\) 同时向上跳，并保持深度一致且二者不相会

具体来说，就是 依次尝试 把 \(a\)，\(b\) 同时向上走 \(k=2^{logn},…,2^1,2^0\) 步，在每次尝试中，若 \(fa[a,k]\ne fa[b,k]\)（即仍未相会），则令 \(x=F[x,k],y = F[y,k]\)

  1. 此时 \(a\)，\(b\) 必定只差一步就相会了 ，它们的父节点 \(fa[a,0]\) 或 \(fa[b,0]\) 就是 \(LCA(a,b)\)
  • 时间复杂度：
    • 对于每个询问，时间复杂度为 \(O(logn)\)
    • 如果有 \(n\) 个点，\(m\) 个询问，总共的时间复杂度为 \(O(mlogn)\)

向上标记法

  • 从 \(x\)​ 向上走到根节点，并标记所有经过的节点。

  • 从 y 向上走到根节点，当第一次遇到已标记的节点时，就找到了 \(LCA(x,y)\)

  • 时间复杂度：

    • 对于每个询问，时间复杂度为 \(O(n)\)​
    • 如果有 \(n\) 个点，\(m\) 个询问，总共的时间复杂度为 \(O(nm)\)

基本不用这种方法

Tarjan-LCA 算法 ★

​ \(Tarjan-LCA\) 算法本质上是 使用并查集对“向上标记法”的优化。它是一个离线算法，需要把 m 个询问一次性读入，统一计算，最后统一输出 - 时间复杂度：\(O(n+m)\)，其中 \(n\) 为节点数量，\(m\) 为询问数量

在深度优先遍历时，将所有树中节点分为三类：

  1. 已经 访问完毕且 回溯 的节点，标记为 \(2\)

  2. 已经开始递归，但尚未回溯的节点，标记为 \(1\)

这些节点就是当前正在访问的节点 \(x\) 以及的 \(x\) 的祖先

  1. 尚未访问的节点，标记为 \(0\)
  • \(tarjan\) 本质是 \(DFS\)​ + 并查集

Bellman-Ford

  • 如果图中存在负权回路，最短路 不一定 存在

    • 如果从 \(1\) 号点到 \(n\) 号点的某条路径上存在负环，则不存在最短路
    • 如果负环不在从 \(1\) 号点到 \(n\) 号点的路径上，就没有影响
    • 如果题目限制了 最短路边的个数（即不能无限转），则一定存在最短路径

  • 在使用 备份数组 避免发生串联更新的情况下，\(Bellman\ Ford\) 算法迭代 \(k\) 次，\(dist[ ]\) 表示从 \(1\) 号点，最多经过 \(k\) 条边 ，走到每个点的最短距离

  • 如果第 \(k\) 次迭代更新了 \(dist[]\)，

    • 且 使用备份数组 避免发生串联更新，则第 \(k\) 次更新的最短路径的边数一定是 \(k\)

    • 不使用备份数组 允许串联式更新，则第 \(k\) 次更新的最短路径的边数一定 \(\ge k\)

    因为我们是用上轮迭代的 \(dist[]\) 更新本轮迭代的 \(dist[]\)，当本轮中 \(dist[]\) 发生更新，说明上轮迭代的 \(dist[]\) 一定发生了更新，而上轮迭代的 \(dist[]\) 更新有说明上上轮迭代的 \(dist[]\) 发生了更新，以此类推说明最短路径的边数为 \(k\)

  • \(Bellman\ Ford\) 算法可以判断图中 是否存在负权回路

    • 迭代 \(n\) 次，如果第 \(n\) 次仍有最短距离更新的话，说明存在一条边数为 \(n\) 的最短路径
    • \(n\) 个点最多有 \(n-1\) 条边，所以一定存在一个负权回路

  • 时间复杂度： \(O(nm)\)​

由于 \(Bellman\ Ford\) 算法只需要在每次循环中，对所有边进行遍历，因此我们可以采用结构体数组存储边的集合即可，不需要存储图

有边数限制的 Bellman-Ford

  • 【模板题： AcWing 853. 有边数限制的最短路 - AcWing】
  • last[] 是备份数组，用于存储上次迭代的结果，用上次迭代的结果更新当前的距离，防止发生串联。为什么需要使用 last[] ?

​ 为了避免如下的串联情况（在一次迭代中更新更多条边）， 在边数限制为 1 条的情况下，节点 3 的距离应该是 3，但是由于串联情况，利用本轮更新的节点 2 更新了节点 3 的距离，所以现在节点 3 的距离是 2

​ 正确做法是用 上轮迭代 中节点 2 的距离 +∞ 和节点 1 的距离 0 来更新节点 3， 再取最小值，所以节点 3 离起点的距离是 3

​ 特别注意：如果没有边数限制（即没有边数限制的 Bellman-Ford），那么串联情况不需要考虑，因为串联情况相当于提前对下轮迭代进行更新，由于没有边数限制所以并不影响

  • 为什么是 dist[n] > 0x3f3f3f3f/2， 而不是 dist[n] > 0x3f3f3f3f

​ 5 号节点距离起点的距离是无穷大，利用 5 号节点更新 n 号节点距离起点的距离，将得到 \(10^9-2\)，虽然小于 \(10^9\)，但并不存在最短路（在边数限制在 k 条的条件下）

没有边数限制的 Bellman-Ford

  • 实际上是“有边数限制的 \(Bellman \ Ford\) 算法”的特殊情况，这是的边数限制是 \(n\) ，但是不需要使用 \(backup[]\) ，因为最短路上最多也就是 \(n\) 个顶点，即使串联也没关系
  • 可以用于判断是否存在负权路

spfa★

​ spfa 算法是 队列优化 的 \(Bellman-Ford\) 算法

  • 定理：对于 所有 的边 \((x,y,z)\) ，都有 \(dist[y]\le dist[x]+z\) 成立 <==> \(dist[]\) 数组是最短路
  • 基本思想： \(Bellman\ ford\)​ 算法会遍历所有的边，但是有很多的边遍历了其实没有什么意义，我们只用遍历那些到源点距离变小的点所连接的边即可，只有当一个点的前驱结点更新了，该节点才会得到更新。因此考虑到这一点，我们创建一个队列，每一次加入距离被更新的结点
  • 时间复杂度：
    • 平均 情况下 \(O(m)\)
    • 最坏 情况下 \(O(nm)\)
    • 如果 SPFA 被出题人卡掉了，改用 堆优化版 dijkstra 算法​

​ 因为 \(SPFA\) 里面每个点进队不止一次，使用循环队列可以节省空间；且如果使用普通队列的话，我们不知道队列长度要开多大，最坏情况下要开 \(n*m\)​（通常会爆空间），为了解决这个问题，我们一般使用循环队列来做，就不需要考虑这些问题了

​ \(SPFA\) 使用 st[] 确保队列中一定不会有重复元素，所以队列当中最多有 \(n\) 个元素，因此如果使用数组模拟循环队列的话，所以我们只需要保证队列长度 \(N\) 严格大于 \(n\) 即可 int q[N]，也可以直接使用 \(C++ \ STL\) 中的循环队列 queue<>

  • 为什么 \(Bellman_ford\) 算法里最后 return -1 的判断条件写的是 dist[n] > 0x3f3f3f3f/2，而 \(spfa\) 算法写的是 dist[n] == 0x3f3f3f3f？

​ 原因在于 \(Bellman\_ford\) 算法会遍历所有的边，因此不管是不是和源点连通的边它都会得到更新；但是 \(SPFA\) 算法不一样，它相当于采用了 \(BFS\)，因此遍历到的结点都是与源点连通的，因此如果你要求的 \(n\) 和源点不连通，它不会得到更新，还是保持的 \(0x3f3f3f3f\)

  • st 数组的作用是什么？

​ 判断当前的点是否已经加入到队列当中了；已经加入队列的结点就不需要反复的把该点加入到队列中了，就算此次还是会更新到源点的距离，那只用更新一下数值而不用加入到队列当中。

​ st 数组是必须的，确保 队列中一定不会有重复元素 ，有 2 方面考虑：

  1. 防止对队列中的重复元素进行更新导致 TLE
  2. 防止队列中重复元素过多，导致队列太长而造成 MLE
  • SPFA 算法看上去和 \(Dijkstra\) 算法长得有一些像但是其中的意义还是相差甚远的:
  1. Dijkstra 算法中的 st[] 数组保存的是当前确定了到源点距离最小的点，且一旦确定了最小那么就不可逆了(不可标记为 true 后改变为 false)；SPFA 算法中的 st[] 数组仅仅只是表示的当前发生过更新的点，且 spfa 中的 st[] 数组可逆(可以在标记为 true 之后又标记为 false)。顺带一提的是 BFS 中的 st[] 数组记录的是当前已经被遍历过的点。
  2. Dijkstra 算法里使用的是优先队列保存的是当前未确定最小距离的点，目的是快速的取出当前到源点距离最小的点；SPFA 算法中使用的是队列(你也可以使用别的数据结构)，目的只是记录一下当前发生过更新的点。
  • \(Bellman ford\) 算法可以存在负权回路，是因为其循环的次数是有限制的，因此最终不会发生死循环。但是 SPFA 算法不可以存在负权回路 ，由于用了队列来存储，只要发生了更新就会不断的入队，因此假如有负权回路，请不要用 SPFA ，否则会死循环
  • 由于 \(SPFA\) 算法是由 \(Bellman\ ford\) 算法优化而来，在最坏的情况下时间复杂度和 \(Bellman\ ford\) 算法一样，时间复杂度为 \(O(nm)\)，但这种情况极为罕见（除非出题人变态，故意把 \(SPFA\) 卡掉），因此如果题目时间允许时，优先考虑使用 \(SPFA\) 求解最短路，如果 \(SPFA\) 真的被卡掉了，我们再用堆优化 \(Dijkstra\) （\(SPFA\) 有点小小万能的感觉）

最小生成树

  • 最小生成树是针对 无向图 的，无向连通图 一定存在最小生成树
  • \(n\) 个顶点的无向连通图，最小生成树的边数为 \(n - 1\)​​​

定理：任意一棵最小生成树一定包含无向图中权值最小的边

反证法。假设无向图 \(G=(V,E)\) 存在一棵最小生成树不包含权值最小的边。设 \(e=(x,y,z)\) 是无向图中权值最小的边。把 \(e\) 添加到树中，\(e\) 会和树上从 \(x\) 到 \(y\) 的路径一起构成一个环，并且环上其他边的权值都比 \(z\) 大。因此, 用 \(e\)​ 代替环上的其他任意一条边，会形成一棵权值和更小的生成树，与假设矛盾。故假设不成立，原命题成立

推论：给定一张无向图 \(G=(V,E)\)，\(n=|V|\)，\(m=|E|\)。从 \(E\) 中选出 \(k<n-1\) 条边构成 \(G\) 的一个生成森林。若再从剩余的 \(m-k\) 条边中选 \(n-1-k\) 条添加到生成森林中，使其成为 \(G\) 的生成树，并且选出的边的权值之和最小，则该生成树一定包含这 \(m-k\) 条边中连接生成森林的两个不连通节点的权值最小的边

Kruskal★

  • \(Kruskal\) 算法是基于上述推论的， \(Kruskal\) 算法总是维护无向图的最小生成森林

最初，可认为生成森林由 \(0\) 条边构成，每个节点各自构成一棵仅包含一个点的树。在任意时刻，\(Kruskal\)​ 算法从剩余的边中选出一条权值最小的，并且这条边的两个端点属于生成森林中两棵不同的树（不连通），把该边加入生成森林， 图中节点的连通情况可以用并查集维护

假设当前 \(Kruskal\) 算法已经循环完第 \(i\) 条边，则已经求出了由前 \(i\) 条边构成的所有连通块

  • 适用于 稀疏图，使用 结构体数组 存储 边的集合 ，不需要存储图

由于 \(Kruskal\) 算法只需要在每次循环中，对 所有边按照边权从小到大 遍历，因此我们可以采用结构体数组存储 边的集合 即可，不需要存储图

  • 时间复杂度：\(O(mlogm)\) ，\(n\) 表示点数，\(m\)​​ 表示边数

  1. \(Kruskal\) 求解最小生成树，同时标记图中的每条边是 树边 还是 非树边：\(O(mlogm)\)

  2. \(BFS\) 预处理得到 \(depth[\ ]\)、\(fa[\ ][\ ]\)、\(d[][]\)

    1. \(depth[i]\) 表示 \(i\) 的深度，\(1\le i \le n\)
    2. \(fa\left[ i,j \right]\) 表示从 \(i\) 开始向上走 \(2^j\) 步到达的节点，\(0\le j \le logn\)

\[ \begin{cases} fa\left [ i,0 \right] = i\ \text{的父节点}\\ \\ fa\left [ i, j \right] = fa\left [ anc , j-1 \right] , 1\le j\le logn, \\ \end{cases} \]

​ (c) \(d1\left[ i,j \right]\) 和 \(d2[i,j]\) 分别表示从 \(i\) 开始向上走 \(2^j\) 步的路径中边权 最大值 和边权 严格次大值 ，\(0\le j \le logn\) \[ \begin{cases} d1\left [ i,0 \right] = i\ \text{到父节点的边权}\\ \\ d1\left [ i, j \right] = max(d1 [i, j-1], d1\left [ fa\left[ i, j-1 \right] , j-1 \right]) , 1\le j\le logn\\ \end{cases} \]

\[ \begin{cases} d2\left [ i,0 \right] =-\infty\\ \\ d2\left [ i, j \right] =\left\{ d1\left [ i, j-1 \right] , d1\left [ anc, j-1 \right] , d2\left [ i, j-1 \right] , d2\left [ anc, j-1 \right] \right\} , 1\leqslant j\leqslant \log n\\ \\ anc = fa\left [ i, j-1 \right] \end{cases} \]

  1. 依次枚举 非树边 \(a\xrightarrow{c}b\)：\(O(mlogn)\)​

    1. 将该边加入树中
    2. \(LCA\) 树上倍增 求解树中从 \(a\) 到 \(b\) 路径上 最大 边权 \(dist1\) 和 严格次大 边权 \(dist2\)：\(O(logn)\)​

正确性证明

  • 定义：
    • 设 \(T\) 为图 \(G\) 的一棵生成树，对于非树边 \(a\) 和树边 \(b\)，插入边 \(a\)，并删除边 \(b\) 的操作记为 \((+a,-b)\)
    • 如果 \(T(+a,-b)\) 后，仍然是一棵生成树，称 \((+a,-b)\) 是 \(T\) 的一个 可行交换
    • 称由 \(T\) 进行一次可行变换所得到的新的生成树集合称为 \(T\)​ 的 邻集
  • 定理：次小生成树一定在最小生成树的邻集中，即一定存在一颗次小生成树，且其与最小生成树只有一条边不同

: star: 该定理对 严格最小生成树 和 非严格最小生成树 都成立

  • 证明：假设存在一个图，其次小生成树与最小生成树至少有 2 条边不同

（1）非严格次小生成树的证明

将最小生成树的边从小到大的顺序排序，找到第一个不在次小生成树中的边假设为 a-> b。则按照 Kruskal 算法，当枚举到 a-> b 这条边时，a 和 b 一定在 2 个不同的连通分量中，次小生成树中不包含这条边，次小生成树中一定存在 从 a 到 b 的其他路径，由于我们是按照从小到大的顺序枚举的所有边，当前 a 和 b 不连通，之后联通了，而且没有使用 a-> b 这条边，则次小生成树中 a 和 b 连通路径上一定用了这条边之后的一条边，即为 c-> d，由于我们是按照从小到大的顺序枚举的所有边，则 c-> d 这条边的权值一定大于等于 a-> b 这条边的权值，将次小生成树去掉 c-> d 加上 a-> b，得到的一定也是一颗新的生成树，且权值之和小于等于原来的次小生成树，且与最小生成树不一样的边数减少了 1，不断重复上述过程，直到次小生成树和最小生成树只差一条边

（2）严格次小生成树证明

将最小生成树的边从小到大的顺序排序，找到第一个不在次小生成树中的边假设为 a-> b。则按照 Kruskal 算法，当枚举到 a-> b 这条边时，a 和 b 一定在 2 个不同的连通分量中，次小生成树中不包含这条边，次小生成树中一定存在 从 a 到 b 的其他路径，由于我们是按照从小到大的顺序枚举的所有边，当前 a 和 b 不连通，之后联通了，而且没有使用 a-> b 这条边，则次小生成树中 a 和 b 连通路径上一定用了这条边之后的一条边，即为 c-> d，由于我们是按照从小到大的顺序枚举的所有边，则 c-> d 这条边的权值一定大于等于 a-> b 这条边的权值

[1] c-> d 这条边的权值等于 a-> b 这条边的权值，则将次小生成树去掉 c-> d 加上 a-> b，得到的仍然是一颗生成树，且与原来的次小生成树权值之和相等，我们就得到了一颗新的次小生成树，且与原来次小生成树不一样的边数减少了 1

[2] c-> d 这条边的权值严格大于 a-> b 这条边的权值，则将次小生成树去掉 c-> d 加上 a-> b，得到的仍然是一颗生成树，但权值之和小于原来的次小生成树权值之和，这与严格次小生成树的定义矛盾

求二分图的最大匹配——匈牙利算法

相关概念

  • 匹配 是“任意两条边都没有公共端点”的 边的集合；最大匹配 是包含边数最多的一组匹配
  • 对于任意一组匹配 \(S\)（\(S\) 是一个边集），属于 \(S\) 的边被称为“匹配边”，不属于 \(S\) 的边被称为“非匹配边”。匹配边的端点被称为“匹配点”，其他节点被称为“非匹配点”。如果在二分图中存在一条连接两个非匹配点的路径 \(path\)，使得非匹配边与匹配边在 \(path\) 上交替出现，那么称 \(path\) 是匹配 \(S\) 的 增广路，也称 交错路

增广路即 非匹配点—非匹配边—匹配边—非匹配边— \(\cdots \cdots\) —匹配边—非匹配边—非匹配点

  • 增广路显然具有以下性质：

    1. 路径边数 \(len\) 是 奇数
    2. 路径上第 \(1,3,5,…,len\) 条边是非匹配边，第 \(2,4,6,…,len-1\) 条边是匹配边（非匹配边的数量比匹配边的数量多 \(1\)）

    正因为以上性质，如果我们把路径上所有边的状态取反，原来的匹配边变成非匹配的，原来的非匹配边变成匹配的，那么得到的新的边集 \(S’\) 仍然是一组匹配，并且匹配边数增加了 \(1\)。进一步可以得到推论：

    二分图的一组匹配 \(S\) 是最大匹配，当且仅当图中不存在 \(S\) 的增广路

差分约束

视频讲解：3.5 差分约束 - AcWing

原理

对于 任意一条 有向边 （假设为 \(j\xrightarrow{c}i\)）

  • 在求完 单源最短路 之后，均存在下面的“三角不等式”：

\[ dist\left [ i \right] \leqslant dist\left [ j \right] +c \]

否则如果存在 \(dist\left[ i \right] > dist\left[ j \right] +c\) ，就一定可以用 \(dist\left[ j \right] +c\) 更新 \(dist\left[ i \right]\)，与单源最短路矛盾

  • 在求完 单源最长路 之后，均存在下面的“三角不等式”：

\[ dist\left [ i \right] \geqslant dist\left [ j \right] +c \]

否则如果存在 \(dist\left[ i \right] < dist\left[ j \right] +c\) ，就一定可以用 \(dist\left[ j \right] +c\) 更新 \(dist\left[ i \right]\)，与单源最长路矛盾

因此，单源最短/长路 问题与 差分约束 问题 等价

注：最短路问题与最长路本质也是一致的

源点条件

⭐ 源点 需要满足的条件：从源点出发，一定可以走到 所有的边

  • 否则：用单源最短路做的话，如果有一条边走不到，则该边对应的不等式就无法满足
  • 某一个点 x[i] 走不到无所谓（某个点走不到表明它不受任何限制，取任何数都可以）
  • 如果从源点出发可以走到所有顶点，则必然也可以到达所有边（反过来不成立，因为可能有孤立点）

应用 1：求可行解

问题

差分约束 可以求解如下不等式组的 一组可行解： \[ x\left [ i \right] <= x\left [ j \right] +c\left [ k \right],\ \ \ \ \ i, j, k = 1,2,...\ \ \ 其中 c [j] 为常数 \]

步骤

  1. 把 \(x\left[ i \right] <=x\left[ j \right] +c\left[ k \right]\) 转化为 \(j\xrightarrow{c_\left[k\right]}i\)
  2. 然后在这个图上 随便 找一个 超级源点，使得该源点一定可以遍历到 所有边
  3. 从 超级源点 求一遍 单源最短路
  4. 求完单源最短路之后：
    1. 存在负环 <=> 不等式无解
    2. 没有负环 <=> \(dist[]\) 即一个可行解

最长路类似，在此不予阐述

应用 2：求最小值

问题

  • 求 每个变量 \(x[i],i=1,2,…\) 的 最小值（使用 最长路）

\[ \left\{ \begin{array}{l} x\left [ i \right] \geqslant x\left [ j \right] +c\left [ j \right]\\ x\left [ k \right] \geqslant c\left [ k \right]\text{或 }x [k] = c [k]\\ \end{array}, i/j = 1,2,... \right. \text{其中}c\left [ 1,2,... \right] , k\text{为常数} \]

  1. 如果 \(x[k]=c[k]\) ，则转化为 \(x[k]\ge c[k],\quad x[k]\le c[k]\)
  2. 多了一个绝对关系不等式 \(x\left[ k \right] \geqslant c\left[ k \right]\)，否则所有变量减去任意常数仍然是不等式的一组解

步骤

  • 建立 超级源点（假设为 \(0\) 号点），则 \(x[0] = 0\) 【绝对关系】

  • 把 \(x[k] \ge c\) 转化为 \(0\xrightarrow{c}k\)，\(x[k] \le c\) 转化为 \(k\xrightarrow{-c}0\)

  • 把 \(x[i]\ge x[j] + c[k]\) 转化为 \(j\xrightarrow{c_\left[k\right]}i\)
  • 求源点 \(0\) 号点到其余所有点的 最长 路径，\(dist[]\) 即为最大值

应用 3：求最大值

问题

  • 求 每个变量 \(x[i],i=1,2,…\) 的 最大值（使用 最短路）

\[ \left\{ \begin{array}{l} x\left [ i \right] \leqslant x\left [ j \right] +c\left [ j \right]\\ x\left [ k \right] \leqslant c\left [ k \right]\text{或 }x [k] = c [k]\\ \end{array}, i/j = 1,2,... \right. \text{其中}c\left [ 1,2,... \right] , k\text{为常数} \]

  1. 如果 \(x[k]=c[k]\) ，则转化为 \(x[k]\ge c[k],\quad x[k]\le c[k]\)
  2. 多了一个绝对关系不等式 \(x\left[ k \right] \leqslant c\left[ k \right]\)

步骤

  • 建立 超级源点（假设为 \(0\) 号点），则 \(x[0] = 0\) 【绝对关系】
  • 把 \(x[k] \le c\) 转化为 \(0\xrightarrow{c}k\)，\(x[k] \ge c\) 转化为 \(k\xrightarrow{-c}0\)

  • 求源点 \(0\) 号点到其余所有点的 最短 路径，\(dist[]\) 即为最小值

原理

欧拉路径与欧拉回路

定义

  • 欧拉路径：也叫“一笔画”路径，指从某一点出发，经过图中的 所有边，且 每条边仅经过一次 的路径
  • 欧拉回路： 起点等于终点 的 欧拉路径

定理

  1. 对于无向图，且 所有边都是连通的，则
    1. 存在欧拉路径的充要条件：度数为奇数的点只能有 0 个或 2 个（起点和终点）
    2. 存在欧拉回路的充要条件：度数为奇数的点只能有 0 个

无向图且度数为奇数的点只有 0 个：此时从任何一个点出发均存在一条欧拉回路

无向图且度数为奇数的点只有 2 个：这两个点一个是起点，一个是终点

  1. 对于有向图，且 所有边都是连通的，则
    1. 存在欧拉路径的充要条件：要么所有点的出度均等于入度；要么除了两个点之外，其余所有点的出度等于入度，剩余的两个点：一个满足出度比入度多 1（起点），另一个满足入度比出度多 1（终点）
    2. 存在欧拉回路的充要条件：所有点的出度均等于入度

无论是有向图还是无向图，存在欧拉路径和欧拉回路的必要条件是，所有的边都是连通的（即存在某一点，使得从该点出发，可以经过图中所有边）

  • AcWing 1123. 铲雪车 - AcWing

分层图

​ 分层图是一种将点拆开，复制多层图，并利用 特殊构造的边（例如权值为 \(0\) 的边）将 各层相连 的建图方法，一般用于 边/点 有 特殊限制 的问题 （如 重复经过次数，多种价值可选，删掉一些边 等）

模板

​ 以 AcWing 2953. 飞行路线 - AcWing 为例：我们把免费操作转化成零边，如果进行免费操作，就进入下一层，蓝边就是零边，建 \(k + 1\) 层图，如果执行免费操作就进入下一层。总共有 \(k + 1\) 层，对应 \(k\) 次操作

​ 由于最多（而非必须）进行 \(k\) 次免费操作，因为问题就等价为求从 第 \(0\) 层的源点（ \(s\) 号节点）到第 \(0\sim k\) 任意一层汇点（ \(t\) 号节点）的最短距离。为了方便求解，我们在每 相邻层的汇点 之间连接一条 权值为 \(0\) 的边，这样问题就等价为求从 \(0\) 层的 \(s\) 号节点到第 \(k\) 层的 \(t\) 号节点的最短距离

当最短路径上的总边数小于 \(k\)（eg：总边数 \(m\) 小于 \(k\)），也就是说可能最短路径在中间某层的汇点就已经达到了 \(0\)，再向下走反而会使总路径增大。为了避免这种情况，我们可以在每两个相邻层的汇点之间连接一条权值为 \(0\) 的边。

正确性证明

只要图中不存在负环，则某一点到另一点的最短路径一定不会经过某条边两次，因此

注意

分层图建立了 \(k+1\) 层图

  • 顶点数是原来的 \(K+1\) 倍
  • 边数是原来的 \(2 * K+1\) 倍

空间复杂度为 \(O(k(n+m))\) ，因此当顶点数 \(n\) 、边数 \(m\) 、层数 \(k\) 较多时，分层图可能会 \(MLE\)

技巧

重边、自环

  • 自环对算法没有影响

  • 重边对适用邻接矩阵求最短路算法有影响

多组测试数据

邻接表初始化

​ 如果题目中有 多组测试用例，每个测试用例都需要重新建立树/图，则每次 只需要对 idx、 h[] 重新初始化，以 AcWing 323. 战略游戏 - AcWing 为例

st[], dist[] 初始化

如果多组测试用例每个测试用例都要使用 \(dijkstra\)，那么切记在 \(dijkstra\) 使用前对 st[], dist[] 初始化

建立新图/反向图

​ 假设原图的 \(1\le N\le 10^5,1\le M\le 10^6\) ，此时求完 SCC 并缩点之后需要建立新图（AcWing 368. 银河 - AcWing、AcWing 1175. 最大半连通子图 - AcWing），或者需要建立原图的反向图（即将所有边反向，AcWing 341. 最优贸易 - AcWing），需要注意以下 3 点

  1. 将邻接表的最大边数 \(M\) 的范围增大到原来的 \(2\) 倍，最大顶点数 \(N\) 不变
  2. 建立 \(2\) 个表头：\(hs[]\) 是原图的表头，\(ht[]\) 是新图的表头，\(e[M], ne[M],w[M],idx\) 可以共用
  3. 加边函数多一个数组参数 \(int\ h[]\)，用于指定是向原图 \(hs[]\) 插入还是向新图 \(ht[]\) 插入

st [] 作用总结

  • \(SPFA\) 中：标记节点是否 入队/在队列中
  • \(dijkstra\)：标记节点的 最短路 是否 已经求出
  • \(DFS\)/\(BFS\)：标记节点是否 已经遍历过
  • \(Targan-LCA\)：标记节点类型——已遍历且回溯、已遍历未回溯、未遍历
  • 匈牙利算法：标记第二个集合中的每个点是否已经被考虑过

TLE

大概率是你忘记初始化 \(h[]\) 数组了，或者多组测试数据你忘记初始化 \(h[]\) 数组 + idx = 0
