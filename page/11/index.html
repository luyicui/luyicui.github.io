会话维持

通过调用get()或post()等方法可以做到模拟网页的请求，但是这实际上是相当于不同的会话，也就是说相当于你用了两个浏览器打开不同的页面。如果第一个请求利用post()方法登录了网站，第二次想获取登录成功后的自己的个人信息，又使用了一次get()方法取请求个人信息，实际上，这相当于打开了两个浏览器，所以是不能成功的获取到个人信息的。为此，需要会话维持，你可以在两次请求时设置一样的 Cookies，但这样很繁琐，而通过 Session 类可以很轻松地维持一个会话。

进度条的含义

输出结果：

我们以以下一行 tqdm 输出为例说明：

  1. 10%|█ |
    • 当前任务完成了 10%。
    • 可视化进度条显示 █。
  2. 1/10
    • 已完成 1 次迭代，总共需要完成 10 次。
    • 目前正在处理第 2 次迭代
  3. 00:27：已完成的迭代次数（这里为 1 次）累计耗时 27 秒。
  4. <04:05：预计剩余时间是 4 分 5 秒。
  5. 27.29s/it：当前每次迭代的指数移动平均耗时为 27.29 秒。

tqdm 使用了指数移动平均（EMA）来平滑显示的速度，因此 s/it 并不是严格基于当前累计耗时和迭代次数计算的，而是对历史耗时进行平滑处理后的结果。

平滑公式如下： \[ \text{smoothed\_s/it} = \alpha \cdot \text{current\_s/it} + (1 - \alpha) \cdot \text{previous\_smoothed\_{s/it}} \]

  • \(\alpha\) 是平滑系数（默认值为 0.3）。
  • current_s/it 是当前最新一个 item 处理的即时时间（即时速率）
  • previous_smoothed_s/it 是上一次的平滑结果。

举例： \[ 7.29\ne \left( 28*60+43 \right) /307 \\ 7.29=0.3*7.77+\left( 1-0.3 \right) *7.08 \]

在构建现代Web应用程序时，性能和并发能力是重要的考量因素。传统的WSGI服务器已无法满足异步Web框架的需求，而ASGI（Asynchronous Server Gateway Interface）成为新一代Web服务器接口的标准。Uvicorn 是一个快速、轻量级的ASGI服务器，支持异步Web框架（如FastAPI和Starlette）以及HTTP/2和WebSocket。Uvicorn 基于 uvloop 和 httptools 构建，具有高性能、易用性和灵活性的特点。

主要功能

  • 支持ASGI协议：完全兼容ASGI标准，可以运行异步和同步的Web应用。

  • 高性能：基于 uvloop 和 httptools，支持高吞吐量和低延迟。

  • 支持HTTP/2和WebSocket：满足现代Web应用的实时通信需求。

  • 热重载：开发模式下支持代码热重载，提升开发效率。

  • 灵活的配置：通过命令行参数或Python代码自定义运行选项。

这些功能使 Uvicorn 成为构建高性能Web应用的理想选择。

typing 模块是 Python 提供的一个标准库，主要用于为函数、变量和类定义类型提示（Type Hints），从而提高代码的可读性和类型安全性。虽然 Python 是动态类型语言，但通过 typing 模块，开发者可以明确指定变量和函数的参数、返回值的类型，帮助 IDE 或静态代码分析工具提供更好的代码提示和错误检查。虽然 Python 本身不会强制执行这些类型检查，但借助 mypy 等工具，可以进行静态类型分析，帮助发现潜在问题。

下面我们再来详细看下 typing 模块的具体用法，这里主要会介绍一些常用的注解类型，如 List、Tuple、Dict、Sequence 等等，了解了每个类型的具体使用方法，我们可以得心应手的对任何变量进行声明了。

Iterable、Iterator

Iterable 和 Iterator 都是 Python 中 typing 模块中的类型别名，用于表示可迭代对象和迭代器。它们的区别在于：

  • Iterable 表示可迭代对象，即可以被 for 循环遍历的对象（如列表、元组、字典、集合等）。Iterable 的类型注解形式为 Iterable[T]，其中 T 表示可迭代对象中元素的类型。Iterable 类型的对象具有 __iter__ 方法，可以使用 iter() 函数来获取一个迭代器。
  • Iterator 表示迭代器，即具有 __next__ 方法的对象，可以通过 next() 函数一个一个地访问元素。例如，对可迭代对象使用 iter() 函数就可以得到迭代器。Iterator 的类型注解形式为 Iterator[T]，其中 T 表示迭代器中元素的类型。迭代器对象可以使用 next() 函数来获取下一个元素，如果没有下一个元素，则会抛出 StopIteration 异常。

简单来说，可迭代对象是一类具有 __iter__ 方法的对象，该方法可以返回一个迭代器对象。而迭代器对象是一类具有 __next__ 方法的对象，该方法返回下一个元素，如果没有下一个元素则抛出 StopIteration 异常。因此，可以使用 iter() 函数将可迭代对象转换为迭代器对象，然后使用 next() 函数逐个访问元素。需要注意的是，迭代器对象是一次性的，即迭代完一次后不能再次迭代，需要重新获取迭代器对象才能再次迭代。

前向引用

在 Python 的类型注解中，使用引号（'Element'）表示类型名表示一个前向引用（forward reference），它告诉 Python 稍后会有一个名为 Element 的类定义，如果后续代码中没有定义 Element 类，类型检查器（如 mypy）或者运行时的某些工具（如果直接解析类型注解）会报错。前向引用通常发生在以下情况下：

  • 类尚未定义时引用其名称：当你在类型注解中引用一个类，而这个类还没有定义时，需要用字符串形式表示该类的名称。这是因为 Python 在解析类型注解时需要知道类的定义，而如果类尚未定义，则会导致解析错误。

  • 避免循环依赖：如果类型注解需要引用当前类本身（即递归类型）或类之间相互引用，用引号可以避免类型检查时报错。

前向引用，带引号

  • 'Element' 用引号表示，这是一种前向引用，告诉解释器稍后会定义一个名为 Element 的类。
  • 这种写法适用于当类 Element 尚未定义的情况。

直接引用类名，不带引号

在不加引号的情况下，Element 必须在函数定义之前已经声明，否则会引发 NameError。

两者的区别

特性 使用 'Element' 直接使用 Element
定义时机 Element 类可以在注解之后定义 Element 类必须在注解之前定义
适用场景 类尚未定义、递归类型等前向引用场景 类已定义时可以直接使用
错误行为 类未定义会导致类型检查器报错 类未定义会导致运行时报错
Python 版本支持 适用于所有版本 适用于所有版本

抽象基类

在Python中，抽象基类（Abstract Base Classes，简称ABC）是一种用于定义接口或抽象方法的工具。它们允许你指定一个类的基本结构，而不必实现所有的细节。通过使用抽象基类，你可以强制子类实现特定的方法，从而确保代码的一致性和可维护性。本文将深入探讨Python中抽象基类ABC的实践技巧与优雅实现。

什么是抽象基类？

抽象基类是一个包含抽象方法（即只有方法签名/声明而没有具体实现）的类，它的主要作用是定义一组接口或规范，而不是提供具体的实现。抽象基类通常用于定义一组类的通用行为，以确保这些类具有相似的接口和行为。

Objects

Cluster

Ray 集群由一个主节点（head node）和多个工作节点（worker nodes）组成。主节点负责集群的资源调度和任务分发，而工作节点负责实际执行任务。通过 ray.init()，可以连接到现有的 Ray 集群，或者启动本地集群以便开发和测试。

@property

python @property 装饰器使一个方法可以像属性一样被使用，而不需要在调用的时候带上() 。接下来我们会深入了解一下我们什么时候需要使用它，并且在什么场景下需要用到它以及如何很好的使用它。

@property简介

你在看review别人代码的时候，可能看到过在方法上添加property 装饰器的场景。不过在深入了解之前，你需要对python中的class 有一定的了解，因为通常我们使用property 装饰器就用在类中。

没有@property会发生的尴尬情况

比如，当你修改self.last 的时候，你会希望self.fullname 会随着self.last 的改变而自动更新。但是self.fullname 并没有更新，这个属性的值就可能误导使用者 。

@property的使用

因此，如果可以将这个方法转化为一个属性，那就可以兼顾两者，不改代码量的基础上实现两种功能。之前我们说过@property 可以使类中的方法，可以像类中属性一样的方式被调用。于是，可以加上一个@property 装饰器来解决这个问题。在方法定义上面加一个@property 装饰器，在不改变原有调用方式的同时，来将一个属性改为一个方法。

  1. 什么时候使用 @property 装饰器呢？当一个B属性的值，是通过另一个A属性得来的。并且这个B属性会随着初始A属性得值得改变而更新
